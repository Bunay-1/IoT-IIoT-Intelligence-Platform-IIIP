# Ръководство за потребителя: Automotive Quality Control

## 1. Какво върши този модул?
Този модул е "сърцето" на качествения контрол. Той комбинира **SPC (Статистически процесен контрол)** за следене на стабилността на производството и **FMEA (Анализ на отказите)** за оценка на рисковете.

## 2. Начин на стартиране
Стартиране на демо версията:
```bash
python automotive_quality_control.py
```
За визуализация използвайте GUI модула:
```bash
python automotive_quality_gui.py
```

## 3. Начин на работа
- **Регистриране на процес**: Дефинирайте име, целева стойност и граници на толеранс (USL/LSL).
- **Добавяне на данни**: Подавайте списък с измервания от производствената линия.
- **FMEA**: Въведете данни за евентуален дефект (тежест, честота, откриваемост).

## 4. Интерпретиране на резултатите
- **Cpk Индекс**:
  - `Cpk > 1.33`: Процесът е изключително стабилен и надежден.
  - `Cpk < 1.0`: Процесът не е способен да поддържа качеството; изисква се корекция.
- **RPN (Risk Priority Number)**:
  - `RPN > 100`: Висок риск. Изисква незабавни превантивни мерки.
  - `RPN < 40`: Нисък риск.
